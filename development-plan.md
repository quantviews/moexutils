# План развития проекта MOEX Utils

## Обзор
Проект фокусируется исключительно на данных Московской биржи (MOEX). Основные цели: получение, сохранение и организация данных акций, облигаций и производных инструментов. Дополнительно — простые расчеты метрик (например, доходность, волатильность, Greeks). Никаких ML-моделей или международных данных — это отдельные проекты.

## Фаза 1: Облигации (3-6 месяцев)
- **Получение данных**: Функции для скачивания котировок облигаций через ISS MOEX API (цены, купоны, сроки).
- **Сохранение**: Аналогично акциям — Parquet-файлы в папке data/.
- **Простые расчеты**: YTM (yield to maturity), duration, convexity.
- **Организация**: Интеграция с существующими скриптами обновления.

## Фаза 2: Производные инструменты (6-9 месяцев)
- **Фьючерсы**: Получение и сохранение данных, объединенная история (roll-over контрактов).
- **Опционы**: Данные по коллам/путам, расчет Greeks (delta, gamma, theta).
- **Сохранение**: Parquet с метаданными (тип контракта, expiration).
- **Организация**: Расширение update_data.py для деривативов.

## Фаза 3: Улучшения инфраструктуры (9-12 месяцев)
- **Кэширование**: Redis или локальный кэш для ускорения запросов.
- **API**: Простой REST API для доступа к данным.
- **Веб-интерфейс**: Базовый дашборд для просмотра и экспорта данных (Flask/Streamlit).
- **Организация**: Улучшение структуры папок, логирование.

## Фаза 4: Дополнительные метрики и аналитика (12-18 месяцев)
- **Метрики**: Расширение для акций (Sharpe ratio, max drawdown), облигаций (credit spread), деривативов (implied volatility).
- **Визуализация**: Простые графики в ноутбуках (matplotlib/seaborn).
- **Организация**: Модулизация кода, тесты (pytest).
